# Functional Requirement Table

The front-end functional requirements currently implemented in the Defend Service Portal will also be applied to the Service Portal GPU Merdeka. Here describe Functional Requirement for Super Admin and Admin User.

**Table 1. Functional Requirement - Super Admin**

| Feature ID | Functional Requirement | Description | Acceptance Criteria |
|-----------|-------------------------|-------------|---------------------|
| DB-SA-001 | Management Location | The Super Admin has the authority to add, update, delete, and manage site the locations use for Deka Box. | can add, update, and delete locations.<br>System must validate unique location names.<br>Success/failure messages are shown after each operation. |
| DB-SA-002 | Configure Deka Box Pricing | The Super Admin has the authority to determine the pricing of Deka Box. | can input or update pricing for Deka Box.<br>Pricing must be saved correctly in the system.<br>Confirmation message appears after saving. |
| DB-SA-003 | Set Quota | The Super Admin has the authority to set limits for Admin Users. These limits include the quota for Deka Box creation and the size in GB that will be used. | can set quota for number and size of Deka Boxes.<br>Quota values must be stored and enforced per user.<br>System notifies success or failure of update. |
| DB-SA-004 | Logs Module | The Superadmin can view and export audit logs of Deka Box creation/deletion, credential access, and user actions. | can filter and view logs by type and date.<br>Export to CSV/PDF is available and accurate.<br>System loads logs without failure or timeout. |
| DB-0005 | Create Module | The Super Admin and User Admin can create a Deka Box by specifying the type Basic, region, volume size, Billing type, and a unique name to identify the Deka Box. | can input required fields: name, type, region, size, billing.<br>Deka Box is created only after all validations pass.<br>Success/failure message is displayed. |
| DB-0006 | Detail Module | The Super Admin and User Admin can view several details including Overview, Access Key, and Settings | can view Overview, Access Key, and Settings tabs.<br>System loads each section without error. |
| DB-0007 | Overview-Add Bucket | Super Admin dan User Admin dapat menambah bucket ke dalam Deka Box yang sudah dibuat dengan hanya memasukkan Bucket Name melalui Layanan Portal Cloudeka | can input bucket name and add to Deka Box.<br>Bucket appears in list if added successfully.<br>Error is shown if name is invalid or duplicate. |
| DB-0008 | Overview-Delete Bucket | Super Admin dan User Admin dapat menghapus bucket yang tersedia pada Deka Box yang sudah dibuat melalui Layanan Portal Cloudeka | can delete selected bucket from Deka Box.<br>Bucket is removed from list upon confirmation.<br>System shows confirmation or failure message. |
| DB-009  | Overview - Add CORS Configuration | Super Admin dan User Admin dapat menambah CORS Configuration melalui Layanan Portal Cloudeka ada bucket yang dipilih. Untuk dapat menambahkan CORS configuration perlu memasukkan Origin, Allowed Methods (GET/PUT/DELETE/POST/HEARD) dan Allowed Headers. Setelah berhasil menambahkan terdapat tombol Apply Changes untuk menyimpan CORS Configuration. | inputs Origin, Methods, Headers.<br>Clicking 'Apply Changes' saves configuration.<br>Success message appears if CORS added properly. |
| DB-0010 | Overview - Delete CORS Configuration | Super Admin dan User Admin dapat menghapus CORS Configuration melalui Layanan Portal Cloudeka yang sudah ditambahkan pada bucket yang dipilih. | can delete selected CORS config.<br>Confirmation message is shown post-deletion. |
| DB-0011 | Manage Access Key - Add | Super Admin dan User Admin dapat menambah kunci akses melalui Layanan Portal Cloudeka yang dapat digunakan untuk mengakses Deka Box. Pada saat menambahkan tersedia kolom Access Key Name dan pengguna yang akan menggunakan. Jika sudah berhasil menambahkan maka tampil Access Key ID dan Secret Access Key | can input Access Key Name and user info.<br>system generates and displays Access Key ID and Secret Key.<br>success notification is shown after creation. |
| DB-0012 | Manage Access Key - Destroy | Super Admin dan User Admin dapat menghapus kunci akses melalui Layanan Portal Cloudeka yang dapat digunakan untuk mengakses Deka Box kecuali akses Creator. | can delete access keys except those owned by the creator.<br>system confirms successful or failed deletion. |
| DB-0013 | Manage Access Key - Acces Key ID | Super Admin dan User Admin dapat menyalin Access Key ID yang tersedia melalui Layanan Portal Cloudeka | can copy the displayed Access Key ID.<br>copy action confirms visually or via message. |
| DB-0014 | Manage Access Key - Visible Secret Key | Super Admin dan User Admin dapat melihat Secret Access Key melalui Layanan Portal Cloudeka dan perlu memasukkan Access Key ID serta Password yang digunakan saat login ke Layanan Portal Cloudeka dan Super Admin dapat menyalin Secret Access Key. | can input Access Key ID and password to reveal Secret Key.<br>revealed Secret Key can be copied securely. |
| DB-0015 | Settings Bucket | Super Admin dan User Admin dapat menghapus semua bucket yang tersedia pada proyek yang ada dalam organisasi melalui Layanan Portal Cloudeka. | can delete all buckets within an organization's project.<br>confirmation is required before deletion. |
| DB-0016 | Increase Size | Super Admin dan User Admin dapat meningkatkan ukuran kuota yang sudah ada melalui Layanan Portal Cloudeka | can increase existing quota size.<br>updated quota must reflect immediately.<br>success message shown after update. |
| DB-0017 | Access Bucket with S3 Browser | Super Admin dan User Admin dapat mengakses bucket yang sudah dibuat di Layanan Portal Cloudeka melalui S3 Browser dengan memiliki tipe akun "S3 Compatible Storage" menambahkan API endpoint, Access Key ID, dan Secret Access Key | can access Deka Box bucket using S3 Browser.<br>must provide API endpoint, Access Key ID, and Secret Access Key.<br>connection is successful only with valid credentials. |
| DB-0018 | Region Configuration | Super Admin dan User Admin dapat menambahkan konfigurasi region pada bucket melalui S3 Browser dan akan dibuat sesuai dengan region yang sudah terdaftar di Layanan Portal Cloudeka. | can configure region for a bucket via S3 Browser.<br>selected region must match one registered in Cloudeka.<br>system must validate and store configuration. |
| DB-0019 | Bucket Policy Configuration | Super Admin dan User Admin dapat menetapkan kebijakan yang akan digunakan pada bucket. Terdapat beberapa kebijakan yang dapat diterapkan diantaranya Create Bucket, DeleteBucket, GetBucketLocation, BucketPolicy, Bucket Tagging, GetBucketVersioning, ListAllMyBucket, ListBucketVersions. | can set one or more policies for a bucket.<br>must be able to apply standard S3 permissions (Create, Delete, List, etc).<br>confirmation shown after policies are saved. |
| DB-0020 | Add Bucket using S3 Browser | Super Admin dan User Admin dapat menambahkan bucket melalui S3 Browser dengan memasukkan nama bucket dan region yang terdaftar pada Layanan Portal Cloudeka. | can input bucket name and region to create new bucket.<br>bucket must appear in S3 Browser after creation.<br>invalid input triggers error notification. |
| DB-0021 | Delete Bucket using S3 Browser | Super Admin dan User Admin dapat menghapus bucket melalui S3 Browser. | can delete bucket through S3 Browser.<br>bucket is removed from list upon successful deletion.<br>system confirms deletion with message. |
| DB-0022 | CORS Configuration using S3 Browser | Super Admin dan User Admin dapat menambah CORS Configuration melalui S3 Browser pada bucket yang dipilih. Untuk dapat menambahkan CORS configuration perlu memasukkan Origin, Allowed Methods (GET/PUT/DELETE/POST/HEARD) dan Allowed Headers. | can input Origin, Methods, and Headers for CORS.<br>S3 Browser applies configuration after submission.<br>confirmation message shown upon success. |
| DB-0023 | Versioning Configuration using S3 Browser | Super Admin dan User Admin dapat menghapus semua bucket yang tersedia pada proyek yang ada dalam organisasi melalui Layanan Portal Cloudeka. | can configure or disable versioning on a bucket.<br>system must reflect versioning status immediately.<br>confirmation displayed after changes applied. |
| DB-0024 | Upload file/Folder using S3 Browser | Super Admin dan User Admin dapat menambahkan berkas ke bucket melalui S3 Browser. | can select and upload file or folder to bucket.<br>uploaded content must appear in the destination path.<br>system confirms completion of upload. |
| DB-0025 | Delete file/folder using S3 Browser | Super Admin dan User Admin dapat menghapus berkas ke bucket melalui S3 Browser. | can select and delete file or folder from bucket.<br>item is removed from view after successful deletion.<br>system confirms deletion result. |
